扩散语言模型（Diffusion Language Models, DLMs）一直在追赶自回归模型。目前表现最好的 DLM 多在离散 token 空间上做扩散（如 MDLM、Duo），而连续空间的 DLM 一直效果欠佳。ELF 的核心主张是：连续 DLM 的落后并非因为语言本质上必须离散建模，而是因为之前的设计选择没有充分发挥连续空间的优势。通过在连续 embedding 空间中使用 Flow Matching，并将离散化推迟到最后一步，ELF 用更少的训练数据和采样步数超过了已有的离散和连续 DLM。

## 前置知识：扩散模型与扩散语言模型

### 扩散模型

扩散模型的核心思想很直观：给数据逐步加噪声直到变成纯随机噪声，然后训练一个神经网络学会"逆过程"——从噪声一步步恢复出干净数据。生成时，从随机噪声出发，反复调用这个去噪网络，最终得到一个有意义的样本。

以图像为例：训练时，取一张图片，逐步往上叠加高斯噪声，经过 $T$ 步后变成完全随机的噪声图。然后训练模型在任意噪声水平 $t$ 下，预测"加了多少噪声"或"原始干净图像是什么"。生成时从纯噪声开始，每一步让模型预测并去掉一点噪声，经过 $T$ 步迭代就能生成逼真的图像。Stable Diffusion、DALL-E 等模型都基于这一框架。

**Flow Matching** 是扩散模型的一种现代表述方式。它不再用"逐步加噪/去噪"的离散步骤视角，而是定义一条从噪声到数据的连续路径。最简单的形式就是线性插值：时刻 $t=0$ 是纯噪声，$t=1$ 是干净数据，中间时刻 $t$ 的状态就是两者的加权混合。模型学习的是这条路径上每一点的"速度场"（即下一瞬间应该往哪个方向走）。Flow Matching 目前被广泛用于 Stable Diffusion 3、FLUX 等图像生成模型。

### 扩散语言模型

文本是离散的 token 序列，不像图像那样天然连续，所以把扩散模型搬到语言上需要额外处理。目前有两条路线：

- **离散扩散**：直接在 token 空间上定义"加噪"过程——比如把 token 随机替换成 [MASK]（类似 BERT 的做法但多步迭代），或替换成随机 token。生成时从全 [MASK] 或全随机序列出发，逐步恢复出有意义的文本。代表方法有 MDLM、Duo 等。
- **连续扩散**：先把 token 通过 embedding 层映射到连续向量空间，然后在向量空间里做标准的加噪/去噪，最后再把向量映射回 token。代表方法有 Diffusion-LM、CDCD 等。

相比自回归模型（GPT 系列）从左到右逐 token 生成，扩散语言模型可以并行处理整个序列，一次性生成/修改所有位置，这在某些场景下（如翻译、摘要、可控生成）有潜在优势。

ELF 属于连续扩散路线，但做了一个关键简化：去噪过程中完全不做 token 级监督，只在最后一步才投影回离散 token。

## 连续 vs 离散：之前的问题在哪

之前的连续 DLM 效果不如离散 DLM，ELF 认为原因在于设计上的保守：

- Diffusion-LM、CDCD 等方法虽然在连续空间做去噪，但在每一步都加入 token 级约束（如 rounding loss、每步 cross-entropy），相当于强迫中间状态时刻"靠近"某个具体 token，限制了连续轨迹的自由度
- Latent Diffusion 类方法（如 LD4LG）在压缩的隐空间中操作，还需要额外训练一个解码器

ELF 的做法是彻底放开中间过程：整个去噪轨迹都在连续 embedding 空间中自由流动，中间步骤的向量不需要对应任何 token，只在最后一步做一次离散投影。

## ELF 的方法设计

![Figure 3](/assets/images/elf/x3.png)
*Figure 3: ELF 训练和推理流程。训练时，token 经编码器变为连续 embedding $x$，加噪得到 $z_t$，模型预测 $\hat{x}$。推理时从高斯噪声出发逐步去噪，仅在最后一步切换为"解码模式"映射回离散 token。*

### 从 token 到连续 embedding

给定 token 序列 $s = [s_1, \ldots, s_L]$，ELF 用一个编码器（默认为预训练的 T5-small encoder）将其映射到连续表示 $x$。编码器只在训练时使用，推理时不需要。

### 在 embedding 空间做 Flow Matching

ELF 采用 Flow Matching 的 rectified flow 形式。加噪过程就是标准的线性插值：

$$z_t = t \cdot x + (1-t) \cdot \epsilon, \quad \epsilon \sim \mathcal{N}(0, \mathbf{I})$$

其中 $t \in [0, 1]$，$z_0$ 是纯噪声（标准高斯），$z_1$ 是干净 embedding。中间任意时刻 $t$ 的状态就是干净数据和噪声的加权混合。训练时 $t$ 从一个 logit-normal 分布中采样（偏向噪声较大的区域），噪声 $\epsilon$ 还会乘以一个系数 2 来增大扰动幅度。

流的速度场为 $v = dz/dt = x - \epsilon$。模型采用 $x$-prediction（预测干净 embedding 而非速度），训练损失为：

$$\mathcal{L}_{\text{MSE}} = \mathbb{E}_{t, x, \epsilon} \frac{1}{(1-t)^2} \|x_\theta(z_t, t) - x\|^2$$

选择 $x$-prediction 有两个原因：(1) 在高维 embedding（768 维/token）上表现更好；(2) 预测干净 embedding 与最终步预测 token 的目标天然对齐，便于权重共享。

### 最后一步做离散化

到 $t=1$ 时，模型切换为"解码模式"：对最终 embedding 通过一个 unembedding 矩阵 $W$ 映射为 logits，再用 cross-entropy 训练：

$$\mathcal{L}_{\text{CE}} = \mathbb{E}_{\tilde{z}} [\text{CrossEnt}(W \cdot x_\theta(\tilde{z}), s)]$$

这里有一个细节：$t=1$ 时 $z_t$ 已经等于干净 embedding $x$ 了，直接拿来训练解码没有意义。所以在训练时，ELF 对最后一步的输入额外做了一次 corruption：$\tilde{z} = p \cdot x + (1-p) \cdot \epsilon$，其中 $p$ 从 logit-normal 分布采样（偏向较大值，即大部分信息保留但有轻微扰动），且不同 token 位置的 $p$ 不同。这样做的目的是让解码器对"不完美的 embedding"有鲁棒性——毕竟推理时去噪器产生的最终 embedding 不可能完全干净。

去噪网络和解码网络共享权重，通过一个二值 mode token（"denoise" 或 "decode"）区分当前模式。训练时两个分支按 80%/20% 的比例混合。推理时只在最后一步执行 argmax 得到 token。

这意味着 ELF 不需要额外的解码器——Flow Matching 的最后一步天然充当了连续到离散的桥梁。

### Classifier-Free Guidance (CFG)

CFG 是图像扩散模型中控制生成质量的标准技术：同时训练有条件和无条件两种模式，推理时将两者的预测做线性外推来增强条件信号。由于 ELF 在连续空间操作，CFG 可以直接应用（离散 DLM 中 CFG 因为缺乏连续量而效果不佳）。

ELF 利用 self-conditioning 作为 CFG 的条件信号：第一次前向传播得到中间预测 $\hat{x}'$，第二次以 $\hat{x}'$ 为条件得到最终预测。为避免推理时额外开销，ELF 采用 training-time CFG 技术，单次前向传播即可得到带 guidance 的输出。

## 实验结果

### 关键消融

![Figure 5](/assets/images/elf/x5.png)
*Figure 5: 设计选择消融。(a) 预训练的上下文 embedding 最优；(b) 共享权重解码与两阶段训练效果相当但更简洁；(c) SDE 采样器在少步时显著优于 ODE。*

几个关键发现：

- **Embedding 选择**：预训练的上下文 embedding（T5 encoder）效果最好，从头训练的 encoder 略差，非上下文 embedding（如单一 embedding 层）更差，可学习 embedding 最差
- **解码策略**：共享权重与两阶段分开训练效果相当，但共享权重能在低困惑度区间延伸得更远，且管线更简单
- **采样器（ODE vs SDE）**：这里解释一下这两个术语。ODE（常微分方程）采样是确定性的——给定同一个初始噪声，每次生成的结果完全相同，就是沿着速度场走一条固定轨迹（用 Euler 方法近似）。SDE（随机微分方程）采样则在每一步额外注入少量随机噪声再去噪，引入了随机性。具体来说，ELF 的 SDE 采样器在每步先把当前状态往回"推"一点（混入噪声、时间回退），再让模型去噪，最后沿速度场前进。参数 $\gamma$ 控制注入噪声的强度，$\gamma=0$ 退化为 ODE。实验显示 SDE 采样器在少步采样（如 32 步）时显著优于 ODE，表明引入适量随机性有助于纠正累积误差——如果某步预测偏了，注入噪声相当于给了模型重新校正的机会

### 模型缩放

![Figure 6](/assets/images/elf/x6.png)
*Figure 6: 模型从 105M (ELF-B) 到 342M (ELF-M) 到 652M (ELF-L)，困惑度-熵前沿持续改善。*

三个尺度（105M、342M、652M）均表现出缩放趋势：更大的模型在相同多样性下质量更高，或在相同质量下多样性更高。不过从图中看，ELF-M 到 ELF-L 的提升幅度明显小于 ELF-B 到 ELF-M——模型大了 6 倍（105M → 652M），困惑度的改善并不算大。论文也只展示了三个点，没有给出连续的 scaling law 拟合曲线，所以现阶段还难以判断 ELF 的缩放效率是否能与自回归模型相当。

即使与经过蒸馏的模型（如 MDLM+SDTT、Duo+DCD、FMLM）对比，ELF 在少步采样下仍然更优，且不需要任何额外的蒸馏训练。

在 WMT14 德-英翻译和 XSum 摘要任务上，ELF 均取得最优结果，超过了自回归 baseline 和所有对比的扩散方法。条件生成的实现方式是将条件序列的干净 embedding 拼接在模型输入前面，训练和推理时都保持不加噪。

## 讨论

**只能做序列生成吗？** 从论文实验看，ELF 验证了三种场景：无条件文本生成（从零生成一段文本）、机器翻译（给源语言序列生成目标语言序列）、摘要（给长文生成短摘要）。这些都是 sequence-to-sequence 任务。原理上，ELF 的条件生成框架（把条件序列的干净 embedding 拼在前面）可以扩展到任何 seq2seq 场景（如对话、改写、代码生成）。但与 GPT 类自回归模型相比，ELF 有一个结构性限制：它需要预先固定生成序列的长度 $L$（训练时 OWT 固定为 1024，翻译固定为 64）。这意味着开放式对话、变长输出等场景下的适用性还不明确。此外，论文没有测试 ELF 在分类、信息抽取等判别式任务上的表现——这些并非其设计目标。

**Scaling 是否明显？** 从 Figure 6 看，ELF-B (105M) → ELF-M (342M) → ELF-L (652M) 的改善存在但不算显著。在 CFG scale=3、SDE 64步的设置下，三个模型的 Gen. PPL 差异并不大（都在 20-30 范围内）。相比之下，自回归 LLM 从 100M 到 1B 通常会有更可观的 perplexity 下降。可能的原因：(1) 训练数据只有 OWT 的 9B token（5 个 epoch），更大模型在小数据上可能已经趋于饱和；(2) 论文提到继续增加训练 token 也没改善，说明瓶颈可能不在模型容量；(3) 作为扩散模型，其 scaling 行为可能与自回归模型不同，需要不同的实验设计来展示。

**为什么连续空间 + 延迟离散化有效？** 之前的连续 DLM 在每一步都加入 token 级约束（如 rounding loss），相当于把连续去噪轨迹绑在离散空间附近。ELF 让去噪过程完全在连续空间自由流动，只在最后做一次离散投影。这给了 Flow Matching 最大的优化自由度——毕竟中间步骤的 embedding 不必对应任何具体 token。

**CFG 在语言扩散中的角色**。CFG 在图像扩散中早已是标配，但在离散 DLM 中因为缺乏连续量而难以应用。ELF 的连续空间设计让 CFG 的迁移变得自然，这可能是其性能优势的重要来源之一——它能在生成质量和多样性之间灵活调节。

**数据效率的来源值得探究**。ELF 用 45B token 就达到了其他方法 500B+ token 的效果，这个差距相当大。论文提到继续增加训练数据也没有进一步提升，暗示可能存在某种信息压缩效率的优势——预训练 T5 encoder 已经编码了语言结构信息，Flow Matching 只需要学习在这个结构化空间中的流动方式。不过这也引出了一个问题：ELF 的性能多大程度上依赖于 T5 encoder 的质量？

**规模化的前景**。目前最大的 ELF-L 也只有 652M 参数，离当前主流 LLM 的规模还有很大差距。缩放曲线看起来增益在减小，加上训练数据饱和的问题，ELF 能否在更大规模上继续保持优势还是未知数。另外序列长度固定为 1024，更长序列上的表现也需要验证。
