> **论文**：DeepSeek-V4: Towards Highly Efficient Million-Token Context Intelligence
> **作者**：DeepSeek-AI
> **模型**：[HuggingFace](https://huggingface.co/collections/deepseek-ai/deepseek-v4)

DeepSeek-V4 系列包含两个 MoE 模型：DeepSeek-V4-Pro（1.6T 总参数，49B 激活）和 DeepSeek-V4-Flash（284B 总参数，13B 激活），均支持 100 万 token 上下文。核心动机是打破 vanilla attention 的二次复杂度瓶颈——随着推理模型和 test-time scaling 的兴起，超长上下文的高效处理成为关键限制因素。V4 通过架构创新实现了效率上的飞跃：在 1M 上下文下，V4-Pro 仅需 DeepSeek-V3.2 的 27% 单 token 推理 FLOPs 和 10% 的 KV cache；V4-Flash 更进一步，只需 10% 的 FLOPs 和 7% 的 KV cache。三项核心技术升级：混合压缩注意力架构（CSA + HCA）、流形约束超连接（mHC）、以及 Muon 优化器。

## 架构升级

![Figure 2](/assets/images/deepseek-v4/fig2-07.png)
*Figure 2：DeepSeek-V4 整体架构——混合 CSA/HCA 注意力层、DeepSeekMoE FFN 层、mHC 残差连接*

DeepSeek-V4 保留了 DeepSeekMoE 框架和 Multi-Token Prediction（MTP）策略，在三个方面做了关键改进。MoE 部分有一个细节变化：将计算 affinity score 的激活函数从 $\text{Sigmoid}(\cdot)$ 改为 $\sqrt{\text{Softplus}(\cdot)}$，并移除了对 routing target node 数量的约束。同时将初始几层的 dense FFN 替换为采用 Hash routing 的 MoE 层（根据输入 token ID 的预定义哈希函数确定目标 expert）。

### Manifold-Constrained Hyper-Connections (mHC)

标准 Hyper-Connections（HC）将残差流宽度扩展 $n_{\text{hc}}$ 倍——残差状态的形状从 $\mathbb{R}^d$ 变为 $\mathbb{R}^{n_{\text{hc}} \times d}$，其中 $d$ 是实际层输入的隐藏维度。HC 引入三个线性映射：输入映射 $A^l \in \mathbb{R}^{1 \times n_{\text{hc}}}$、残差变换 $B^l \in \mathbb{R}^{n_{\text{hc}} \times n_{\text{hc}}}$、输出映射 $C^l \in \mathbb{R}^{n_{\text{hc}} \times 1}$，更新公式为：

$$X^{l+1} = B^l X^l + C^l \mathcal{F}^l(A^l X^l)$$

其中 $\mathcal{F}^l$ 是第 $l$ 层（如 MoE 层），其输入输出维度仍为 $d$，所以扩展的残差宽度不影响内部层设计。HC 将残差宽度与实际隐藏维度解耦，以极小的计算开销（$n_{\text{hc}} \ll d$）提供互补的缩放维度。但 HC 的问题是**多层堆叠时频繁出现数值不稳定**。

mHC 的核心改进是将残差映射矩阵 $B^l$ 约束到**双随机矩阵流形**（Birkhoff 多面体）上：

$$B^l \in \mathcal{M} \triangleq \{ M \in \mathbb{R}^{n \times n} \mid M\mathbf{1}_n = \mathbf{1}_n, \mathbf{1}_n^T M = \mathbf{1}_n^T, M \geq 0 \}$$

这个约束保证谱范数 $\|B^l\|_2 \leq 1$，使残差变换非扩张，从而在前向传播和反向传播中增强数值稳定性。而且双随机矩阵集合在乘法下封闭，保证了深层 mHC 堆叠的稳定性。输入和输出变换也通过 Sigmoid 约束为非负有界（$A^l = \sigma(\tilde{A}^l)$，$C^l = 2\sigma(\tilde{C}^l)$），避免信号抵消。

**动态参数化**：三个映射的参数由动态（输入相关）和静态（可学习偏置）两部分组成。给定输入 $X^l$，先展平并 RMSNorm 归一化，然后通过可学习的投影矩阵生成未约束的原始参数，加上静态偏置，再乘以可学习的 gating factor（初始化为小值）。以输入映射为例：

$$\tilde{A}^l = \alpha_l^{\text{pre}} \cdot (\hat{X}^l W_l^{\text{pre}}) + S_l^{\text{pre}}$$

其中 $\alpha_l^{\text{pre}}$ 是 gating factor，$W_l^{\text{pre}}$ 是投影矩阵，$S_l^{\text{pre}}$ 是静态偏置。

残差映射的约束通过 **Sinkhorn-Knopp 算法**实现：先对 $\tilde{B}^l$ 取指数保证正性，得到 $M^{(0)} = \exp(\tilde{B}^l)$，然后迭代进行列归一化和行归一化 $M^{(t)} = \mathcal{T}_r(\mathcal{T}_c(M^{(t-1)}))$，迭代 $t_{\max} = 20$ 次收敛到双随机矩阵。

### 混合注意力：CSA + HCA

这是效率提升的核心。V4 设计了两种高效注意力架构，以交错（interleaved）方式混合使用，大幅降低长文本场景的计算成本。

#### Compressed Sparse Attention (CSA)

![Figure 3](/assets/images/deepseek-v4/fig3-10.png)
*Figure 3：CSA 核心架构——KV 压缩 → Lightning Indexer top-k 选择 → 共享 KV Multi-Query Attention，滑动窗口补充局部信息*

CSA 的核心思路是"先压缩，再稀疏"，分为以下步骤：

**1. KV 压缩**：将每 $m$ 个 token 的 KV 压缩为一个 entry。具体地，对输入隐藏状态 $H \in \mathbb{R}^{n \times d}$，先计算两组 KV entry $C^a, C^b \in \mathbb{R}^{n \times c}$（$c$ 为 head 维度）及其对应的 compression weights $Z^a, Z^b$：

$$C^a = H \cdot W^a_{KV}, \quad C^b = H \cdot W^b_{KV}, \quad Z^a = H \cdot W^{aZ}, \quad Z^b = H \cdot W^{bZ}$$

然后采用**双向重叠压缩**：第 $i$ 个压缩块同时使用当前块的 $C^a_{mi:m(i+1)-1}$ 和**前一个块**的 $C^b_{m(i-1):mi-1}$，共 $2m$ 个 entry。通过 Softmax 归一化的 compression weights（附加可学习位置偏置 $B^a, B^b \in \mathbb{R}^{m \times c}$）做加权求和：

$$C_i^{\text{Comp}} = \sum_{j=mi}^{m(i+1)-1} S_j^a \odot C_j^a + \sum_{j=m(i-1)}^{mi-1} S_j^b \odot C_j^b$$

注意虽然每个压缩块使用 $2m$ 个 entry，但 $C^b$ 的索引与前一个块的 $C^a$ 索引重叠，所以实际压缩率是 $\frac{1}{m}$。这种重叠设计让相邻压缩块之间有信息过渡，避免了硬边界带来的信息丢失。

**2. Lightning Indexer 稀疏选择**：对压缩后的 KV entry 做 top-k 选择。Indexer 用低秩方式生成 query：先做 down-projection 得到压缩向量 $c_t^Q = h_t \cdot W^{DQ}$，再 up-projection 得到 indexer queries。Index score 的计算用了 ReLU 激活和多头加权求和：

$$I_{t,s} = \sum_{h=1}^{n_h^I} w_t^{I,h} \cdot \text{ReLU}(q_t^{I,h} \cdot K_s^{\text{IComp}})$$

ReLU 的使用使得 score 天然非负，未被选中的块对最终注意力没有贡献。选出 top-k 个压缩块后，只对这些块做核心注意力。

**3. 共享 KV Multi-Query Attention**：每个压缩 KV entry 同时充当 key 和 value（即 key = value = $C^{\text{Comp}}$），用 MQA 方式计算注意力。Attention query 也通过低秩方式从同一个压缩向量 $c_t^Q$ 生成（与 indexer query 共享 down-projection）。

**4. 分组输出投影**：由于 $c \cdot n_h$ 很大，直接投影到 $d$ 维的计算量太高。V4 将 $n_h$ 个输出分成 $g$ 组，每组先投影到较小的中间维度 $d_g$（$d_g < c \cdot n_h / g$），再投影到最终输出。

#### Heavily Compressed Attention (HCA)

![Figure 4](/assets/images/deepseek-v4/fig4-12.png)
*Figure 4：HCA 核心架构——更重的压缩（$m' \gg m$），无稀疏选择，直接 dense attention，滑动窗口补充局部信息*

HCA 的思路是"极度压缩，但保留 dense attention"：

- 使用更大的压缩率 $m' \gg m$，将每 $m'$ 个 token 压缩为一个 entry
- **不使用重叠压缩**（与 CSA 不同），每个压缩块只使用自己块内的 token
- **不做稀疏选择**，所有压缩 KV entry 都参与 dense attention
- 同样采用共享 KV MQA 和分组输出投影
- Query 也通过低秩方式生成

HCA 的设计逻辑是：极度压缩已经把序列长度缩减到很小（$n/m'$），所以即使是 dense attention 计算量也很少。它捕捉全局粗粒度信息，与 CSA 的局部细粒度信息形成互补。

#### 共同附加设计

两种注意力层都包含以下技术：

**Query/KV 归一化**：核心注意力前对每个 head 的 query 和 KV entry 做 RMSNorm，避免注意力 logits 爆炸。

**部分 RoPE**：只在最后 64 维上应用 Rotary Positional Embedding。由于 KV entry 同时做 key 和 value，原始注意力输出会携带绝对位置编码。为解决这个问题，对核心注意力输出也在最后 64 维上施加位置为 $-i$ 的 RoPE，使最终输出携带相对位置编码。

**滑动窗口分支**：为保持因果性，每个 query 只 attend 到前面已完成的压缩块，无法看到当前块内其他 token 的信息。同时最近的 token 通常与 query 最相关。因此额外引入滑动窗口分支：为每个 query token 生成最近 $n_{\text{win}}$ 个 token 的未压缩 KV entry，与压缩 KV 一起参与核心注意力。

**Attention Sink**：设置可学习的 sink logits $\{z'_1, ..., z'_{n_h}\}$，$\exp(z'_h)$ 被加到注意力分母中：

$$s_{h,i,j} = \frac{\exp(z_{h,i,j})}{\sum_k \exp(z_{h,i,k}) + \exp(z'_h)}$$

这允许每个 query head 调整其注意力总分不等于 1，甚至接近 0，为模型提供了"不 attend to 任何信息"的选项。

#### 效率分析

KV cache 采用混合精度存储：RoPE 维度使用 BF16，其余维度使用 FP8，将 KV cache 大小减半。Lightning Indexer 的注意力计算使用 FP4 精度。V4 还比 V3.2 选择了更小的 attention top-k 值，提升短中长文本的效率。

综合来看，在 1M 上下文下，V4 的 KV cache 约为标准 BF16 GQA8（head dimension 128）基线的 **2%**。即使与已经很高效的 V3.2 相比，V4-Pro 也只需 27% 的 FLOPs 和 10% 的 KV cache，V4-Flash 更是只需 10% 的 FLOPs 和 7% 的 KV cache。此外，V4 的 routed expert 参数使用 FP4 精度，虽然在当前硬件上 FP4×FP8 的峰值 FLOPs 与 FP8×FP8 相同，但在未来硬件上理论上可以再快 1/3。

### Muon 优化器

V4 用 Muon 替代 AdamW 训练大部分模块（2D 权重矩阵）。算法流程：

1. 计算梯度 $G_t$
2. 累积动量 $M_t = \mu M_{t-1} + G_t$
3. 对 Nesterov 动量 $\mu M_t + G_t$ 做 Hybrid Newton-Schulz 迭代（近似矩阵极分解）得到 $O'_t$
4. Rescale 更新的 RMS：$O_t = O'_t \cdot \sqrt{\max(n,m)} \cdot \gamma$
5. 应用 weight decay 和更新：$W_t = W_{t-1} \cdot (1-\eta\lambda) - \eta O_t$

对 1D 参数（embedding、normalization、bias、gating 等）回退到 AdamW，因为 Muon 的正交化操作假设权重矩阵至少是 2D 的。

V4 对 Muon 的改进：
- **分组 Newton-Schulz**：实际模型中的权重矩阵通常不是方阵。V4 沿较长维度分组以减小长宽比，提高收敛性
- **更大的 rescaling factor**：$\gamma$ 从 0.01 提升到 0.03，实验表明对训练更有利
- **高效分布式实现**：在大规模 MoE 训练中，expert 参数通过 Expert Parallelism 分片在不同 GPU 上。V4 采用 ZeRO-1 分片动量 buffer，并在 Newton-Schulz 迭代中穿插 allreduce 操作，实现了高效的分布式 Muon

## 基础设施

论文在工程上的投入非常重，这一节的篇幅甚至超过了架构描述。

### MoE 融合 kernel

![Figure 5](/assets/images/deepseek-v4/fig5-16.png)
*Figure 5：EP 方案对比——V4 的 fused kernel 实现了计算、通信、内存访问的完全 overlap*

设计并实现了**单个 fused kernel** 来处理整个 MoE 模块，完全 overlap 计算、Expert Parallelism 通信和内存访问。传统实现中 EP 通信涉及 all-to-all dispatch 和 combine 操作，会造成 GPU 空闲等待。V4 的 fused kernel 将这些操作与 expert 计算交织在一起，消除了通信瓶颈。

### TileLang

采用 TileLang（一种 DSL）开发高效 kernel，在开发效率和运行性能间取得平衡。TileLang 提供 tile-level 编程抽象：开发者在 tile 粒度上编写计算逻辑，由编译器自动处理底层优化。关键特性包括：
- **L1 cache 手动管理**：开发者可以显式控制共享内存的使用，避免自动管理的低效
- **TMA (Tensor Memory Accelerator) 硬件加速**：利用 Hopper GPU 的 TMA 单元做异步数据搬运
- 用 TileLang 实现的 kernel（如 MLA attention、MoE fused kernel）在性能上接近甚至超过手写 CUDA kernel

### Batch-invariant & Deterministic kernels

保证不同 batch 大小、不同并行配置下的 **bitwise 可复现性**。这对以下场景至关重要：
- **Debug**：确保同一输入在不同配置下得到完全相同的输出
- **RL 训练正确性**：RL 的 rollout 和 training 通常使用不同的并行度，如果 forward pass 结果不一致会引入偏差
- **评估一致性**：不同部署配置下的评估结果完全可比

为此，V4 避免使用 nondeterministic 的 atomic 操作，改用确定性的 reduction 策略。

### FP4 量化感知训练

MoE expert 权重和 indexer QK 路径使用 FP4（MXFP4）精度，减少内存和计算开销。训练和推理的精度策略有所不同：
- **训练时**：通过无损 FP4→FP8 反量化复用现有 FP8 混精框架（FP32 master weights），无需修改反向传播流程
- **推理时**：直接使用原生 FP4 权重
- FP4 量化从预训练阶段就已开始，属于**量化感知训练**（QAT），而非事后量化

### 训练框架

**Muon 高效实现**：ZeRO-1 分片动量 buffer + 中间结果 allreduce。由于 expert 参数分片在不同 GPU 上，Newton-Schulz 迭代需要全局信息，V4 在迭代中穿插通信操作。

**mHC 低成本实现**：mHC 的三个线性映射引入额外的内存和计算开销。V4 通过 recomputation（前向不保存中间结果，反向时重新计算）和 fused kernel 来降低成本。

**Contextual Parallelism**：处理压缩注意力的长上下文并行。分两阶段：第一阶段并行计算 KV 压缩，第二阶段在压缩后的短序列上做 attention。

**扩展自动微分**：实现了 tensor-level 的 activation checkpointing，允许对每个 tensor 独立控制是保存还是重计算，比传统的 layer-level checkpointing 更灵活。

### 推理框架

![Figure 6](/assets/images/deepseek-v4/fig6-23.png)
*Figure 6：DeepSeek-V4 的 KV cache 布局——CSA compressed KV、HCA compressed KV、滑动窗口 token-level KV 三种异构 cache*

**异构 KV cache 结构**：CSA 和 HCA 各自有自己的 compressed KV cache，另外还有滑动窗口的 token-level KV cache。三种 cache 的数据格式、精度、大小各不相同，需要统一管理。

**On-Disk KV Cache**：对于 1M token 上下文，即使压缩后的 KV cache 也可能很大。V4 支持将 KV cache 存储到磁盘，在需要时异步加载。

**Shared-Prefix 复用**：在 chat 场景中，多个请求通常共享相同的 system prompt。对于压缩后的 KV entry，如果 prefix 恰好对齐到压缩块边界，可以直接跨请求共享，避免重复计算。这对部署效率非常重要——system prompt 经常占据输入的大部分 token。

## 预训练

### 数据

论文提到数据构造但未详细展开。V4-Flash 训练 32T token，V4-Pro 训练 33T token。相比 V3 的 14.8T token 有大幅增加。数据涵盖多样化的高质量来源。

### 训练设置

两个模型的关键配置：

| | V4-Flash | V4-Pro |
|:---|:---|:---|
| 总参数 | 284B | 1.6T |
| 激活参数 | 13B | 49B |
| 架构 | MoE | MoE |
| 训练 Token 数 | 32T | 33T |

两模型都原生支持 1M 上下文。上下文长度扩展采用两阶段方法，MTP loss 权重在大部分训练中设为 0.3，在 learning rate decay 开始时改为 0.1。

### 训练稳定性

训练 trillion 级 MoE 模型面临显著的稳定性挑战。论文发现 loss spike 与 MoE 层的 outlier 密切相关，routing 机制本身会加剧 outlier 的产生。两项关键技术：

**Anticipatory Routing（预见性路由）**：解耦主干网络和路由网络的同步更新。在 step $t$ 时，用当前参数 $\theta_t$ 做特征计算，但用**历史参数** $\theta_{t-\Delta t}$ 计算路由索引。实践中，在 step $t - \Delta t$ 时提前获取 step $t$ 的数据，"预见性"地计算并缓存路由索引。工程优化包括：精心编排 pipeline 执行与 EP 通信 overlap，将额外开销控制在约 20%；自动检测机制在 loss spike 发生时触发短回滚并激活 Anticipatory Routing，稳定后恢复标准训练。最终以可忽略的额外训练开销避免了 loss spike，且不损害模型性能。

**SwiGLU Clamping**：对 SwiGLU 的线性分量 clamp 到 $[-10, 10]$，gate 分量上界 cap 到 10。这有效消除了 outlier 并稳定训练过程，且不影响性能。

### 预训练评估

在统一内部评估框架下对比三个 base model：

V4-Flash-Base（13B 激活 / 284B 总参）已在**大多数 benchmark 上超过 V3.2-Base**（37B 激活 / 671B 总参），尽管激活参数和总参数都小得多。这说明架构改进、数据质量和训练优化带来的收益超过了参数量的差距。V4-Flash-Base 在世界知识和长上下文任务上优势尤为明显。

V4-Pro-Base（49B 激活 / 1.6T 总参）进一步全面领先，在知识密集型评估上取得了巨大提升（SimpleQA-Verified 从 28.3 提升到 55.2，FACTS Parametric 从 27.1 提升到 62.6），同时在推理、代码和长上下文上也均有提升，是 DeepSeek 系列中最强的基础模型。

## 后训练

后训练采用两阶段范式：先独立训练领域专家，再通过 On-Policy Distillation (OPD) 合并到统一模型。**关键变化是 OPD 完全替代了 V3.2 中的 mixed RL 阶段。**

### 领域专家训练

每个目标领域（数学、代码、Agent、指令跟随等）独立训练一个专家：base model → SFT（高质量领域数据）→ RL（GRPO + 领域特定 reward）。以下是几个值得关注的设计。

#### 推理模式分级

V4-Pro 和 V4-Flash 都支持三种推理 effort 模式：

| 模式 | 特征 | 典型场景 | 响应格式 |
|:---|:---|:---|:---|
| Non-Think | 快速直觉响应 | 日常任务、低风险决策 | `</think>` summary |
| Think High | 有意识的逻辑分析，较慢但更准确 | 复杂问题求解、规划 | `<think>` ... `</think>` summary |
| Think Max | 推理能力的极限发挥 | 探索模型推理上界 | system prompt 指令 + `<think>` ... `</think>` summary |

每种模式在 RL 训练时使用不同的 length penalty 和 context window，导致不同的输出 token 长度。Think Max 模式在 system prompt 开头注入特殊指令：

> *Reasoning Effort: Absolute maximum with no shortcuts permitted. You MUST be very thorough in your thinking and comprehensively decompose the problem to resolve the root cause, rigorously stress-testing your logic against all potential paths, edge cases, and adversarial scenarios.*

#### Generative Reward Model (GRM)

RL 训练中一个核心问题是如何为模型输出提供奖励信号。对于"容易验证"的任务（如数学、代码），可以用规则 verifier 或测试用例直接判对错；但对于"难以验证"的任务（如开放式写作、复杂指令跟随、安全拒绝的质量），传统做法是通过 RLHF 训练一个 scalar reward model（标量打分模型），这需要大量人工标注的偏好数据。

V4 在后训练中**完全弃用了传统的 scalar reward model**，转而采用 Generative Reward Model (GRM) 来处理所有难以验证的任务。其核心设计有以下几个要点：

**Rubric-guided RL 数据**：为每类 hard-to-verify 任务设计评分 rubric（评判准则），基于这些 rubric 构造 RL 训练数据。rubric 相当于将模糊的"好不好"拆解为可操作的评判维度。

**Actor 即 Judge——同一个模型同时做生成和评判**：这是最关键的设计决策。V4 不训练一个独立的 reward model，而是直接对 actor 网络（即 policy model 自身）施加 RL 优化，让它在生成能力之外同时习得评判能力。换言之，GRM 不是一个外部的打分器，而是 actor 网络的一个"内置功能"。模型在生成回复时使用生成能力，在评估轨迹质量时使用评判能力。

**推理能力融入评估过程**：由于评判和生成共享同一个模型，模型的内部推理能力（包括 chain-of-thought、知识检索等）自然融入评估过程。相比传统 scalar RM 只输出一个分数，GRM 可以通过生成式推理来解释为什么给出某个评判，这使得评分更加稳健——模型不只是在做模式匹配式的打分，而是在"思考"后做出评判。

**少量标注即可泛化**：论文称该方法只需少量多样化的人工标注即可达到良好效果，因为模型利用自身的逻辑推理能力来泛化到新的复杂任务，而不是像传统 scalar RM 那样完全依赖标注数据中的统计模式。

**我的一些思考**：

GRM 的"actor 即 judge"设计在概念上非常吸引人，但也引发几个值得深思的问题：

**1. Self-play 的 reward hacking 风险**。当模型同时做生成和评判时，存在一个微妙的循环：模型可能学会生成那些"自己觉得好"而非"真正好"的输出。这本质上是一种 reward hacking——策略和奖励函数是同一个模型，策略优化可能悄悄地同时"贿赂"了奖励函数。论文没有讨论如何防范这种退化。一个可能的缓解手段是 rubric 本身提供了外部约束——模型的评判不是完全自由的，而是要依据预定义的 rubric，这在一定程度上锚定了评判标准。但 rubric 的覆盖面和粒度是否足以防止所有形式的 reward hacking 仍是开放问题。

**2. 与 "Constitutional AI" 思路的关系**。GRM 让模型根据 rubric 自我评判，这与 Anthropic 的 Constitutional AI（RLAIF）有相似之处——都是用模型自身（或类似模型）的判断替代人工标注。区别在于 CAI 用一个独立的模型做 judge，而 GRM 让同一个模型兼任。这种统一可能带来效率优势（不用维护独立 RM），但也失去了 judge 和 actor 之间的"对抗性张力"。

**3. 对 scalar RM 的彻底替代意味着什么**。V4 完全弃用 scalar RM 是一个强信号——说明 DeepSeek 团队认为在他们的规模和场景下，scalar RM 的局限性（信息瓶颈、分布外泛化差、对复杂多维质量的粗糙建模）已经大于其优势（训练简单、推理快）。这与近期社区对 scalar RM 的反思一致：当任务复杂度超过一定阈值后，把丰富的质量判断压缩到一个标量分数会丢失太多信息。GRM 通过生成式推理保留了评判的"过程"和"理由"，信息量远大于一个分数。

**4. 工程上的成本**。GRM 的评判需要模型做一次完整的生成式推理（而非 scalar RM 的单次 forward pass），这意味着 RL 训练中每次 reward 计算的成本大幅增加。论文没有报告 GRM 与传统 scalar RM 在训练效率上的对比。在百万 token RL 的场景下，这个额外成本可能很显著。

#### Quick Instruction

Chatbot 场景中，在生成响应前需要执行多个辅助任务（是否触发搜索、意图识别、权威性分类等）。传统做法用独立的小模型处理，需要冗余 prefill。Quick Instruction 的做法是将专用特殊 token 直接追加到输入序列末尾，每个 token 对应一个辅助任务：

| Token | 功能 |
|:---|:---|
| `<\|action\|>` | 判断是否需要搜索 |
| `<\|title\|>` | 生成对话标题 |
| `<\|query\|>` | 生成搜索 query |
| `<\|authority\|>` | 分类来源权威性需求 |
| `<\|domain\|>` | 识别用户 prompt 领域 |
| `<\|extracted_url\|>` / `<\|read_url\|>` | 判断是否需要读取 URL |

直接复用已计算的 KV cache，完全避免冗余 prefill，搜索 query 生成和领域判断等任务可以并行执行。显著降低 TTFT，消除了维护额外小模型的工程开销。

### On-Policy Distillation (OPD)

多个领域专家作为 teacher，统一模型作为 student，优化 reverse KL loss：

$$\mathcal{L}_{\text{OPD}}(\theta) = \sum_{i=1}^{N} w_i \cdot D_{\text{KL}}(\pi_\theta \| \pi_{E_i})$$

其中 $w_i$ 是每个 expert 的权重，由其相对重要性决定。计算 reverse KL 需要从 student $\pi_\theta$ 采样训练轨迹以维持 on-policy 学习。底层逻辑是统一策略 $\pi_\theta$ 根据当前任务上下文选择性地向相关专家学习（数学任务学数学专家、代码任务学代码专家）。通过 logits 级别的对齐，将不同 expert 的知识整合到统一参数空间，**绕过了传统权重合并或混合 RL 中常见的性能退化**。该阶段使用超过十个 teacher 模型覆盖各个领域。

关键工程决策：采用 **full-vocabulary logit distillation** 而非 token-level KL 近似。先前工作通常将 full-vocabulary KL loss 简化为每个 token 位置的 KL 估计，复用 RL 框架计算。虽然资源高效，但**梯度估计方差大、训练不稳定**。V4 保留完整的 logit 分布计算 reverse KL loss，获得更稳定的梯度估计和更忠实的知识蒸馏。

为使 full-vocabulary OPD 在规模上可行，做了大量工程：
- **Teacher 权重 offload**：所有 teacher 权重卸载到集中式分布式存储，在 teacher forward pass 时按需加载，使用 ZeRO-like 参数分片减轻 I/O 和 DRAM 压力
- **Hidden states 缓存与 logit 重建**：直接物化 $|V| > 100k$ 的全词表 logits 即使写入磁盘也不可行。V4 只缓存 teacher 最后一层 hidden states，训练时重新过 prediction head 重建 logits，重计算开销可忽略
- **Teacher head 流式加载**：训练样本按 teacher index 排序，确保每个 mini-batch 中同时只有一个 teacher head 在 GPU 上，最多一个 teacher head 驻留 device memory
- **异步加载**：所有参数和 hidden states 的加载/卸载在后台异步进行，不阻塞计算关键路径
- **专用 TileLang kernel**：加速 teacher 和 student logits 间精确 KL 散度的计算，减少动态内存分配

### RL/OPD 基础设施

#### Preemptible Rollout

GPU 集群采用全局抢占式任务调度器。V4 实现了 **token 粒度的 Write-Ahead Log (WAL)**：每生成一个 token 就追加到该请求的 WAL 中。Preemption 时暂停推理引擎并保存未完成请求的 KV cache；恢复时用 WAL 和 KV cache 继续解码。即使遭遇致命硬件错误，也可用 WAL 中的 token 重新 prefill 重建 KV cache。

关键正确性考量：**不能从头重新生成未完成的请求**。因为短回复更容易在中断中"存活"，从头重新生成会引入长度偏差，使模型倾向于产生更短的序列。如果推理栈是 batch-invariant 和 deterministic 的，理论上可以用一致的随机种子重新生成，但这仍然有重新运行 decoding 的额外成本，远不如 token 粒度 WAL 高效。

#### 百万 Token RL 优化

- Rollout 数据格式分解为轻量 metadata 和重量级 per-token fields
- Metadata 先加载用于全局 shuffle 和 packing layout 计算
- Per-token fields 通过 shared-memory data loader 加载（消除节点内数据冗余），消费后立即释放
- On-device mini-batch 数量根据负载动态确定，在计算吞吐和 I/O overlap 间权衡

#### DSec 沙箱平台

DeepSeek Elastic Compute (DSec) 是面向 agentic AI 的生产级沙箱平台，由三个 Rust 组件构成：API 网关 (Apiserver)、节点代理 (Edge)、集群监控 (Watcher)，基于 3FS 分布式文件系统。单集群管理**数十万并发沙箱实例**。

四种执行基底统一接口：
- **Function Call**：无状态调用，pre-warmed 容器池消除冷启动
- **Container**：Docker 兼容，EROFS on-demand loading 实现高效镜像组装
- **microVM**：基于 Firecracker，VM 级隔离，适合安全敏感的高密度部署
- **fullVM**：基于 QEMU，支持任意客户操作系统

切换基底只需改一个参数，所有基底共享命令执行、文件传输、TTY 访问的统一 API。

镜像加载优化：容器用 3FS-backed EROFS layers 做 overlay lowerdirs，文件 metadata 在挂载时就绑定到本地磁盘，数据块按需从 3FS 加载；microVM 用 overlaybd 磁盘格式，只读 base layer 在 3FS 上跨实例共享，写入到本地 copy-on-write layer，支持链式快照和毫秒级恢复。

高密度部署优化：消除虚拟化环境中的重复 page-cache，应用内存回收实现安全超量分配；缓解容器运行时的 spinlock 争用，降低每沙箱 CPU 开销。

Trajectory logging 与 preemption-safe resumption：每个沙箱维护全局有序 trajectory log，持久记录每个命令调用及结果。作用：(1) 客户端快速恢复——preemption 时沙箱资源保留，恢复时回放已完成命令的缓存结果，避免非幂等操作的重复执行；(2) 细粒度溯源——每个状态变化可追踪来源；(3) 确定性重放——任何历史会话可以忠实复现。

## 思考

DeepSeek-V4 最核心的贡献是证明了在 MoE 架构上，通过压缩注意力（CSA + HCA）可以将百万 token 上下文从"理论上支持"变成"效率上可行"。KV cache 降到标准基线的 2%、对比前代推理 FLOPs 降到 27%，这不是边际改进而是数量级的变化。这使得长上下文不再是奢侈品，而是可以日常使用的能力——论文标题中的"routinely support"正是这个意思。

CSA 和 HCA 的互补设计很精巧：CSA 通过中等压缩 + 稀疏选择保留重要的局部细节；HCA 通过极度压缩 + dense attention 捕捉全局粗粒度信息。两者交替使用让模型在不同层对信息做不同粒度的处理。但值得注意的是，KV compression 本质上是有损的——论文没有分析在什么情况下压缩会导致关键信息丢失。MRCR 在 128K 后性能下降可能部分归因于此。

mHC 的双随机矩阵约束是一个优雅的解决方案：通过限制残差映射的谱范数来保证信号传播稳定，同时利用双随机矩阵集合在乘法下封闭的性质保证深层堆叠的稳定性。但 Sinkhorn-Knopp 迭代 20 次的计算开销值得关注——论文通过 recomputation 和 fused kernel 来缓解，但这仍然是额外的复杂度。

后训练的 Specialist → OPD 范式取代了 mixed RL，是一个重要的方法论变化。传统的 mixed RL 在多目标间容易产生冲突，而 OPD 让每个领域专家独立优化到极致，再通过 logits 级蒸馏合并。Full-vocabulary logit distillation 比 token-level KL 近似保留了更完整的知识分布，但工程复杂度极高——十多个 trillion 级 teacher 的调度、hidden states 缓存与 logit 重建、按 teacher index 排序以控制 GPU 内存。论文在这些工程细节上的描述可能是目前公开文献中对大规模 OPD 实现最详细的记录。

Generative Reward Model 的引入也值得关注——弃用 scalar RM，让 actor 自身兼任 judge，利用模型内部推理能力做评估。这减少了对大规模人工标注的依赖，但自评估的可靠性和可能的 reward hacking 问题论文没有讨论。

从结果看，V4 在开源模型中已经全面最优，但论文也坦承与 frontier 闭源模型有 3-6 个月的差距。不过 V4 的效率优势意味着它可以更 aggressively 地做 test-time scaling——相同的 FLOPs 预算下，V4 能处理更长的上下文、生成更多的推理 token。Codeforces rating 3206（超过 GPT-5.4 的 3168）就是这种效率优势转化为性能优势的一个例子。
